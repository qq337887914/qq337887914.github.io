10.给定一个链表，判断链表中是否有环

给定一个链表，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 true 。 否则，返回 false 。

进阶：

你能用 O(1)（即，常量）内存解决此问题吗？

示例 1：

输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点。
示例 2：

输入：head = [1,2], pos = 0
输出：true
解释：链表中有一个环，其尾部连接到第一个节点。
示例 3：

输入：head = [1], pos = -1
输出：false
解释：链表中没有环。

提示：

链表中节点的数目范围是 [0, 104]
-105 <= Node.val <= 105
pos 为 -1 或者链表中的一个 有效索引 。

输出：返回 null

如图，假设：有环时，头节点的位置到入口点的位置长度为X，入口点到相遇点的位置为L，环的长度为C。

fast的速度是slow的两倍，则有，

2(X+L) = X+nC+L；

即X=nC-L；

也就是说，当两者在第一次相遇点相遇了之后，将slow重新从起点，fast从相遇点，两者以相同的速度开始走，必定再次在入口点相遇。

综上，也就是说有环时，当fast以2单位的速度，slow以1单位的速度，两者同时从起点出发，第二个相遇点就是入口点。

入门杂记

1、JDK，JRE，JVM三者的关系是什么？

答：

JDK：Java开发工具包，目前使用最多的就是jdk1.8。

JRE：是Java的运行环境。

JVM：Java虚拟机。

调用顺序：我们使用开发工具（eclipse，IDEA等）编写好我们的程序，在powershell中使用javac命令就是编译过程，也就是使用JDK的过程；使用javac命令后，生成的.class文件在JRE上运行，通过JVM解析，最后映射到我们的操作系统上。

2.小知识

  • 一个字节8位，int型变量是4个字节，即int型变量是32位，首位为符号位，1为负，0为正，最大值为：2^31;

  • double类型8个字节，可以精确表示小数点后16位；float4个字节，可以精确表示小数点后6位；初始化 eg：float f = 12.5f;初始化float型变量后必须加上’f’，因为小数默认为double类型。
  • java当中有8种基本的数据类型：byte，short，int，long，float，double，char，boolean；
